# moTF 카카오 알림톡 템플릿 풀세트

작성일: 2026-07-05

## 기본 원칙

- 알림톡은 예약, 주문, 결제, 환불, 채팅, 문의, 정산 등 정보성 안내에만 사용한다.
- 광고, 이벤트, 혜택, 홍보성 문구는 넣지 않는다.
- 승인 후 문장 구조를 바꾸면 재심사가 필요할 수 있으므로 변수명과 흐름을 고정한다.
- 긴 URL은 본문에 넣지 않고 웹링크 버튼으로 처리한다.
- 채팅 알림은 같은 대화방/수신자 기준 약 5분 단위로 묶어 과도한 발송을 막는다.
- 관리자 알림에서는 새 회원가입, 입금 완료 예약 알림은 제외한다.

## 버튼 기준

| 버튼명 | 연결 대상 |
| --- | --- |
| 예약/주문내역 보기 | 이용자 마이페이지 이용 내역 |

| 주문내역 보기 | 이용자 마이페이지 주문 상세 |

| 채팅 확인 | 이용자 또는 사장님 채팅 화면 |
| 예약 확인 | 사장님 예약 상세 |
| 주문 확인 | 사장님 주문 상세 |

## 심사 전 확인

- 각 템플릿의 변수명은 DB payload와 일치해야 한다.
- 버튼 URL은 `motf.co.kr` 또는 `motfowner.co.kr`의 실제 접속 가능한 주소로 등록한다.
- 사장님 수락/취소 버튼은 즉시 처리하지 않고 확인 화면을 거치게 한다.
- 알림톡 딜러사에 따라 변수 표기법이 `#{변수명}`에서 다른 형태로 바뀔 수 있다.
- 실제 카카오 알림톡 발송은 딜러사 계약, 카카오 채널, 템플릿 승인, 템플릿 코드 입력이 끝난 뒤 mock 발송부를 교체해 진행한다.
